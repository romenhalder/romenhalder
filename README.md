## 💼 Featured Projects

### 🍞 **Bakery Inventory Management System**
> *Enterprise-grade full-stack solution for bakery operations with POS, inventory tracking, supplier management, and intelligent alerting*

**Live Repo:** [bakery-inventory-management](https://github.com/romenhalder/bakery-inventory-management)

**Problem Solved:**
Bakeries lack centralized systems for real-time inventory tracking, expiry management, POS billing, and supplier coordination — leading to stockouts, wastage, and operational chaos.

**Technical Highlights:**
- ✅ **POS Module** — Dynamic product search, shopping cart, payment methods (Cash/UPI/Card), auto-generated PDF invoices
- ✅ **Inventory System** — Real-time stock tracking (current/reserved/available), 5 transaction types (Stock In, Out, Adjustment, Return, Wastage), batch management
- ✅ **Smart Alerts** — Auto-generated alerts for out-of-stock, low-stock, reorder points, expiring/expired products with background scheduler
- ✅ **Supplier Management** — Directory with contact info, FSSAI numbers, active/inactive status, searchable interface
- ✅ **Role-Based Access** — 3 user roles (Admin, Manager, Employee) with granular permissions across 15+ features
- ✅ **Analytics Dashboard** — Sales KPIs, top-selling products, stock health, PDF/CSV exports with jsPDF
- ✅ **Advance Cake Booking** — Custom cake catalog with booking, partial/full payment options, kanban-style tracking

**Tech Stack:**
- **Backend:** Java 17, Spring Boot 3.1.6, Spring Security 6.x, JWT (JJWT 0.12.x), MySQL 8.0, Hibernate 6.x
- **Frontend:** React 19, Redux Toolkit, React Router 7, Tailwind CSS 4, Axios, jsPDF
- **Architecture:** MVC Pattern, Clean Separation of Concerns (Controller → Service → Repository), Custom DTOs, Global Exception Handling

**Metrics:**
- 80+ flavor options across 9 categories
- 22+ bakery brands integrated
- Real-time alerts with background scanning
- Role-based dashboard for 3 user types
- PDF & CSV export capability

### 🏛️ **Civic Issue Reporting System**
> *Smart India Hackathon 2025 Finalist — Crowdsourced platform connecting citizens, administrators, and field workers for municipal issue resolution*

**Live Repo:** [civic-issue-reporting](https://github.com/romenhalder/civic-issue-reporting)

**Problem Solved:**
Citizens lack efficient channels to report municipal issues (potholes, broken streetlights, water supply) and track resolution progress. Administrators struggle with issue triage, worker assignment, and SLA management.

**Technical Highlights:**
- ✅ **Citizen Portal** — Report issues with GPS coordinates, photo attachments, voice notes, real-time status tracking
- ✅ **Authentication** — JWT-based secure auth with Voter ID verification, separate user & admin login flows
- ✅ **Admin Dashboard** — Issue viewing by category, status, ward, priority filtering with SLA deadline tracking
- ✅ **Worker Management** — Worker profile creation, issue assignment with priority levels, task tracking
- ✅ **Issue Categorization** — 5 categories (Road, Street Light, Dustbin, Water Supply, Others) with custom priority logic
- ✅ **SLA & Priority** — Automated SLA deadline calculation (15 days), priority classification based on age (7-day threshold)
- ✅ **Status History** — Full audit trail of status changes with comments and timestamps
- ✅ **Analytics** — Admin dashboard with issue statistics, resolution metrics, geographic clustering

**Tech Stack:**
- **Backend:** Java 21, Spring Boot 3.5.5, Spring Security, JWT (jjwt-api, jjwt-impl), PostgreSQL, Spring Data JPA
- **Architecture:** Modular design with config, controller, service, repository, entity, DTO, security packages
- **Database:** PostgreSQL with 5 core entities (User, Admin, Issue, Worker, IssueStatusHistory)
- **Security:** BCrypt password encryption, CORS configuration, input validation with Jakarta Beans

**Metrics:**
- 5 issue categories
- 3 user roles (User, Admin, Worker)
- Automated SLA tracking
- Real-time status notifications
- Geographic clustering by municipal area/ward
